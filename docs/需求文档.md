# html2md 需求文档

## 概述

html2md 是一个将 Wiki 类 HTML 页面转换为 Obsidian 兼容 Markdown 文档的工具。支持本地 HTML 文件和远程 URL 两种输入方式，针对不同 Wiki 站点（Wikipedia EN/JP、Fandom、ZeldaWiki）提供可配置的转换策略。

## 产品形态

- **核心库**：Python 实现，提供 HTML → Markdown 的转换引擎
- **CLI 工具**：命令行接口，支持单文件/批量转换
- **Web 界面**：后续实现，当前版本不包含

## 功能需求

### F1. 输入源

| ID | 需求 | 说明 |
|----|------|------|
| F1.1 | 本地 HTML 文件 | 读取磁盘上的 HTML 文件进行转换 |
| F1.2 | 远程 URL | 输入 URL，程序自动下载 HTML 后进行转换 |
| F1.3 | 自动识别输入类型 | 程序根据输入参数自动判断是文件路径还是 URL |
| F1.4 | 批量转换 | 支持一次传入多个文件/URL，如 `html2md convert ./test/*.html` |
| F1.5 | URL 输入保留 HTML | 通过 URL 下载的 HTML 文件保存在 MD 输出目录下，便于后续策略更新后重新转换 |

### F2. 核心转换

#### F2.1 文本与结构

| ID | 需求 | 说明 |
|----|------|------|
| F2.1.1 | 正文提取 | 根据站点策略提取主要文章内容，排除导航栏、侧边栏等 |
| F2.1.2 | 标题层级 | 保持原文标题层级结构，转换为 Markdown heading |
| F2.1.3 | 页面标题 | 转换后 MD 文件包含 `# 标题` 和来源行（如 "From Wikipedia, the free encyclopedia"） |
| F2.1.4 | 文本格式 | 粗体、斜体、列表等基础 Markdown 格式对应转换 |
| F2.1.5 | 目录（TOC） | 直接删除，不保留 |

#### F2.2 链接

| ID | 需求 | 说明 |
|----|------|------|
| F2.2.1 | Wiki 内部链接 | 保留完整 URL（如 `https://en.wikipedia.org/wiki/Hideo_Kojima`），点击在浏览器中打开 |
| F2.2.2 | 外部链接 | 保留原始 URL，正常转换 |
| F2.2.3 | 消歧义提示（Hatnote） | 保留为斜体文本 + 链接（如 "For the 2000 Game Boy Color game, see *Metal Gear: Ghost Babel*."） |
| F2.2.4 | Main article 引导链接 | 保留为普通文本链接 |

#### F2.3 图片

| ID | 需求 | 说明 |
|----|------|------|
| F2.3.1 | 外部图片（远程 URL） | 保留为 `[![](缩略图URL)](原始图片URL)` 格式，由 Obsidian 在线加载显示 |
| F2.3.2 | Base64 内嵌图片 | 提取图片数据，保存到同目录下的 `/img` 文件夹，MD 中使用相对路径引用 |
| F2.3.3 | Base64 图片格式 | 自动检测图片格式（PNG/JPEG/GIF 等），使用正确扩展名 |

#### F2.4 表格

| ID | 需求 | 说明 |
|----|------|------|
| F2.4.1 | 普通表格 | 转换为 Markdown 表格 |
| F2.4.2 | 合并单元格 | 对 rowspan/colspan 进行拆分，原始内容放置在合并范围的第一个（左上）单元格中，拆分出的其他单元格留空 |
| F2.4.3 | Infobox 信息框 | 转为 Markdown 表格，保留全部字段 |
| F2.4.4 | Navbox 导航模板 | 删除，不保留 |

#### F2.5 引用系统（核心功能）

这是最核心的功能需求。Wiki 页面有两种引用类型：

- **Notes**：说明性注释，角标为字母（a, b, c, d...）
- **References**：正式引用来源，角标为数字（1, 2, 3...）

| ID | 需求 | 说明 |
|----|------|------|
| F2.5.1 | 角标转 Obsidian footnote | 正文中的角标转换为 Obsidian 兼容的 `[^id]` 格式 |
| F2.5.2 | 数字角标命名 | 按页面显示的数字，命名为 `ref_N`（如页面上显示 `[1]` → `[^ref_1]`） |
| F2.5.3 | 字母角标命名 | 按页面显示的字母，命名为 `note_X`（如页面上显示 `[a]` → `[^note_a]`） |
| F2.5.4 | Footnote 内容 | 仅包含引用原文文本，不包含外部链接 |
| F2.5.5 | 双向跳转 | 点击角标跳到页面底部 footnote，点击 footnote 的返回箭头回到正文位置（Obsidian 原生支持） |
| F2.5.6 | Notes 与 References 分开展示 | 分别放在 `## Notes` 和 `## References` 两个 section 下，footnote 定义放在各自对应的 section |
| F2.5.7 | Footnote 去重 | 同一引用被多次使用时，Obsidian 的 `[^id]` 可重复引用，底部定义只需一份 |

### F3. 站点策略

| ID | 需求 | 说明 |
|----|------|------|
| F3.1 | 内置策略 | 预置四种 Wiki 站点策略：Wikipedia EN、Wikipedia JP、Fandom、ZeldaWiki |
| F3.2 | YAML 配置文件 | 每个站点策略以 YAML 文件形式保存在 `sites/` 目录下，用户可新建或修改 |
| F3.3 | 自动站点检测 | URL 模式优先（按域名匹配 → 策略）；本地文件回退到 HTML Meta 标签检测；均失败时提示用户手动 `--strategy` 指定 |
| F3.4 | 策略配置项 | 见下方"站点策略配置结构" |

### F4. CLI 接口

```
html2md convert <input...> [options]
```

| 参数 | 说明 |
|------|------|
| `<input...>` | 一个或多个输入（文件路径或 URL），至少一个必填 |
| `--output <dir>` | 输出目录（可选，默认与源 HTML 同目录） |
| `--strategy <name>` | 手动指定站点策略（可选，默认自动检测） |
| `--strict` | 严格模式：遇到解析错误立即中止（默认宽松模式） |

### F5. 错误处理

| ID | 需求 | 说明 |
|----|------|------|
| F5.1 | 宽松模式（默认） | 跳过无法解析的部分，继续转换，完成后汇总所有警告信息 |
| F5.2 | 严格模式（--strict） | 遇到错误立即中止，输出具体错误位置和原因 |
| F5.3 | HTML 下载失败 | URL 无法访问时给出明确错误信息，不生成任何文件 |

### F6. 输出

| ID | 需求 | 说明 |
|----|------|------|
| F6.1 | MD 文件命名 | 与源 HTML 文件同名（仅扩展名改为 .md） |
| F6.2 | 输出位置 | 默认与源 HTML 同目录；支持 `--output` 指定其他目录 |
| F6.3 | 图片文件夹 | base64 提取的图片放在 `输出目录/img/` 下 |
| F6.4 | 外部链接区域 | 保留 `## External links` 和 `## Further reading` 为 MD 链接列表 |

## 站点策略配置结构

每个站点策略 YAML 文件包含以下配置项：

### 内容提取

```yaml
content:
  main_selector: ".mw-parser-output"       # 正文容器 CSS 选择器
  title_selector: "#firstHeading"           # 页面标题选择器
  remove_selectors:                          # 需要删除的元素
    - ".mw-editsection"                      # [edit] 编辑链接
    - ".hatnote"                             # 消歧义提示（已单独处理）
    - ".navbox"                              # 底部导航模板
    - "[role='navigation']"
    - ".mw-jump-link"                        # 辅助跳转链接
```

### 引用系统

```yaml
citations:
  superscript_selector: ".reference"         # 角标元素选择器
  cite_href_regex: "#cite_note-(.+)"         # 从 href 提取引用 ID
  notes:
    section_selector: ".mw-references-wrap"  # Notes 区域容器
    item_selector: "li"                      # 单条 note 选择器
    item_id_attr: "id"                       # ID 属性名
    group_value: "Note"                      # group 属性对应的值（区分 Notes 组）
  references:
    section_selector: ".mw-references-wrap"  # References 区域容器
    item_selector: "li"                      # 单条引用选择器
    item_id_attr: "id"                       # ID 属性名
    group_value: ""                          # 空表示默认引用组
```

### 特殊元素

```yaml
elements:
  infobox_handling: "table"                  # info 框处理方式：table | remove
  toc_handling: "remove"                     # 目录处理：remove | keep
  heading_offset: 0                          # 标题层级偏移
  template_selectors:                        # 站点特有模板
    - ".notice-template"
    - ".ambox"
```

### 链接与编码

```yaml
links:
  base_url: "https://en.wikipedia.org"       # 相对链接补全基址
  keep_external_links: true                  # 外部链接
  external_links_sections:                   # 需保留的链接区域
    - "#External_links"
    - "#Further_reading"

encoding: "utf-8"                            # 页面编码
```

## 支持的站点

| 站点 | 标识符 | URL 模式 | 备注 |
|------|--------|----------|------|
| Wikipedia English | `wikipedia_en` | `en.wikipedia.org` | 引用使用 cite_note 系统 |
| Wikipedia Japanese | `wikipedia_jp` | `ja.wikipedia.org` | 同上，注意编码和日文角标 |
| Fandom Wiki | `fandom` | `*.fandom.com` | 引用和模板结构不同 |
| ZeldaWiki | `zeldawiki` | `zeldawiki.net` 等 | 基于 MediaWiki，与 Wikipedia 类似 |

## 非功能需求

| ID | 需求 |
|----|------|
| NF1 | 转换后的 MD 在 Obsidian 中可正常渲染和跳转 |
| NF2 | 转换速度：单个 1MB 以内的 HTML 文件在 3 秒内完成转换 |
| NF3 | 站点策略配置文件格式为 YAML，编码 UTF-8 |
| NF4 | CLI 在 Windows 环境下运行（PowerShell / Git Bash） |

## 转换示例

### 角标转换

**输入（HTML）：**
```html
<sup id="cite_ref-9" class="reference">
  <a href="#cite_note-9">[5]</a>
</sup>
```

**输出（MD 正文）：**
```markdown
...text...[^ref_5]
```

**输出（MD 底部 References）：**
```markdown
[^ref_5]: Kasavin, Greg (October 2, 2000). "Metal Gear Solid (PC) review". *GameSpot*. Archived from the original on May 18, 2007. Retrieved July 7, 2007.
```

### 合并单元格

**输入（HTML）：**
```html
<td rowspan="2">A</td><td>B</td>
<tr><td>C</td></tr>
```

**输出（MD）：**
```markdown
| A   | B   |
|     | C   |
```

### 图片

**外部图片（保留链接）：**
```markdown
[![](https://upload.wikimedia.org/wikipedia/en/thumb/3/33/Metal_Gear_Solid_cover_art.png/250px-Metal_Gear_Solid_cover_art.png)](https://en.wikipedia.org/wiki/File:Metal_Gear_Solid_cover_art.png)
```

**Base64 图片（提取到本地）：**
```markdown
![](img/image_001.png)
```
